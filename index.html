类文件结构

发表于 2018-09-21 | 分类于 JVM

1.魔数和Class文件版本

每个class文件前四个字节称为魔数，就是用来确定该文件是否能被虚拟机所接受的class文件。

第5和第6个字节表示次版本号，第7和第8字节表示主版本号。

2.常量池

紧接着主次版本号之后的就是常量池入口，可理解为class文件的资源库。主要用来存放两类常量：字面量和符号引用。字面量比较接近java语言层面的常量概念，如文本字符串、final修饰的常量等。而符号引用属于编译原理方面的概念，主要包括三类常量：类和接口的全限定名、字段的名称和描述符、方法的名称和描述符。

3.访问标志

常量池之后的两个字节代表访问标志，用于识别一些类或接口层次的访问信息，包括：这个Class是类还是接口；是否定义为public类型；是否定义为abstract类型；如果是类，有没有被final修饰等。

4.类索引、父类索引和接口索引集合

访问标志之后就是类索引、父类索引和接口索引集合。类索引用于确定这个类的全限定名，父类索引用于确定这个类的父类的全限定名，接口索引集合用于描述这个类实现哪些接口。

5.字段表集合

描述接口或类中声明的变量。

6.方法表集合

描述接口或类中声明的方法。

7.属性表集合

描述某些场景专有的信息。

垃圾收集

发表于 2018-09-20 | 分类于 JVM

1.概述

垃圾收集针对的就是java堆和方法区。程序计数器、栈的内存分配和回收都具备确定性的，内存随着方法结束或线程结束而回收，就不用再考虑回收的问题了。而java堆和方法区不一样，例如一个接口的多个实现类需要的内存可能不一样，也就是说我们只有在程序运行时才知道创建哪些对象，所以这部分内存分配和回收都是动态不确定的，就需要我们去考虑回收的问题。

2.判断对象是否存活

主要有两种方式：引用计数法和可达性分析算法

2.1引用计算法

原理：给对象中添加一个引用计数器，每当一个地方引用它，计数器就加1；有一个引用失效，就减1；任何时刻只要计数器值为0就判定对象死亡。

弊端：解决不了对象间相互循环引用的问题。

2.1可达性分析算法

原理：通过一系列称为“GC Roots”的对象作为起始点，从这些节点开始向下搜索，搜索所走过的路径称为引用链，当一个对象到GC Roots没有任何引用链相连时，就说明该对象不可用。

3.垃圾收集算法

3.1标记-清除算法

原理：先标记，再清除。

不足：一个是效率问题，标记和清除的效率都很低；一个是空间问题，内存零散化严重。

3.2复制算法

原理：将可用内存按容量分成大小相等的两块，每次只使用其中一块，当这块内存使用完，就将还存活的对象复制到另一块内存，再把使用过的内存一次清理掉。

优点：效率高，不会产生内存碎片。

缺点：将内存压缩为原来的一半。

3.3标记-整理算法

原理：先标记，然后让所有存活对象都向一端移动，最后清理掉端边界以外的内存。

优点：解决了复制算法在对象存活率较高时要多次复制所造成的效率降低的问题，也不需要额外的空间进行分配担保。

3.4分代收集算法

根据对象存活周期划分内存。一般把java堆分为新生代和老年代，再根据各年代的特点采取不同的回收算法，如新生代存活率低，可用复制算法；老年代存活率高，可用标记清除或标记整理。

4.内存分配

大多数情况，对象在新生代Eden区分配。当Eden区没有足够内存进行分配时，只要老年代的连续空间大于新生代对象总大小或历代晋升的平均大小，就进行一次Minor GC（发生在新生代的GC），否则将执行一次Full GC（发生在老年代的GC）；大对象一般指那种很长的字符串以及数组，会直接进入老年代；长期存活对象将进入老年代。

JVM自动内存管理机制

发表于 2018-09-19 | 分类于 JVM

1.JVM运行时数据区域

1.1程序计数器

程序计数器可看作当前线程所执行的字节码的行号指示器，其实就是告诉JVM执行到字节码文件哪一行了。多线程环境中，每次线程切换后都能恢复到正确的执行位置，靠的就是这个计数器，这也解释了为什么程序计数器必须是线程私有的。另外，它也是唯一一个不会发生内存溢出异常的区域。

1.2虚拟机栈

就是为JVM执行java方法服务的。一个java方法执行的同时会创建一个栈帧用于保存局部变量表、操作数栈、动态链接、方法出口等信息。一个java方法从调用到执行结束，就对应着一个栈帧在虚拟机栈中入栈到出栈的过程。另外，如果线程请求的栈深度大于JVM所允许的深度，就会报StackOverFlowError；如果JVM可以动态扩展，但扩展时却申请不到足够的内存，就会报OutOfMemoryError。

1.3本地方法栈

同虚拟机栈，区别就是本地方法栈为JVM执行native方法服务的。

1.4堆

就是用来存放对象实例的。java堆可以处于物理上不连续的内存空间，只要逻辑连续即可。另外，如果堆内存不足以完成实例分配，且无法扩展，就会报OutOfMemoryError。

1.5方法区

存储JVM加载的类信息、常量、静态变量等数据。方法区无法满足内存分配需求时，会报OutOfMemoryError。

1.6运行时常量池

方法区的一部分。存放各种字面量和符号引用。常量池无法申请到内存时就会报OutOfMemoryError。

2.对象的创建

JVM执行到一条new指令时，

s1：检查指令参数能否在常量池中找到一个类的符号引用，并检查这个符号引用代表的类是否已被JVM加载、解析和初始化过。如果没有，就先执行相应的类加载过程。

s2：为新生对象分配内存。（有两种分配方式，指针碰撞和空闲列表。处理分配内存时可能出现的线程安全问题有两种方法，1.CAS+失败重试，2.本地线程分配缓冲。）

s3：JVM将分配到的内存空间初始化为零值。

s4：JVM设置对象（如这个对象是哪个类的实例）。

s5：把对象按照程序员的意愿进行初始化。

3.对象的内存布局

可分为三块区域：对象头（header）、实例数据（Instance Data）、对齐填充（Padding）。

3.1对象头

包含两部分信息。第一部分用于存储对象自身的运行时数据，如哈希码、GC分代年龄、锁状态标志、线程持有的锁、偏向线程ID等；第二部分是类型指针，JVM就是通过这个指针确定这个对象是哪个类的实例。

3.2实例数据

就是代码中定义的各种类型的字段内容。

3.3对齐填充

JVM自动内存管理机制要求对象大小必须是8字节的整数倍，而对象头正好是8字节的整数倍，所以实例数据也必须是8字节整数倍，如果不是，就要对齐填充。

4.对象的访问方式

两种主流访问方式：句柄和直接指针。

句柄访问最大优势就是reference中存储的是稳定的句柄地址，当对象移动时只需改变句柄中的实例数据指针，而reference本身不需要修改；

直接指针访问最大优势就是节省了一次指针定位的时间开销。

乐观锁和悲观锁

发表于 2018-09-18 | 分类于 java面经

乐观锁：

概念：乐观锁是指操作数据库时(更新操作)，想法很乐观，认为每次的操作不会导致冲突，所以操作数据时不加锁，操作完成后，再判断是否有冲突。

实现：主要有版本号机制或CAS操作两种。

​ 1.版本号机制：前提是操作的数据库表有version字段，每次更新操作，都会将version字段值加1。一次完整的更新操作分为两步：（以更新状态为例）

s1.查出更新行信息，包含version字段。

select (status, version) from table where id = #{id};

s2.更新行信息，version字段加1。

update table set status = #{status}, version = version+1 where id=#{id} and version=#{version};

注：#{version}的值就是第一步查询出来的version值。

如果在更新期间有其他线程更新了该行，就会导致本次更新失败。

​ 2.CAS操作：即compare and swap 或者 compare and set，涉及到三个操作数，数据所在的内存值，预期值，新值。当需要更新时，判断当前内存值与之前取到的值是否相等，若相等，则用新值更新，若失败则重试，一般情况下是一个自旋操作，即不断的重试。

悲观锁：

概念：悲观锁是指操作数据库时(更新操作)，想法很悲观，认为每次的操作都会导致冲突，所以操作数据时都会加锁。

实现：共享锁和排它锁

​ 1.共享锁【S锁】：又称读锁，若事务T对数据对象A加上S锁，则事务T可以读A但不能修改A，其他事务只能再对A加S锁，而不能加X锁，直到T释放A上的S锁。这保证了其他事务可以读A，但在T释放A上的S锁之前不能对A做任何修改。

sql实现：在执行语句后加”lock in share mode”。

​ 2.排他锁【X锁】：又称写锁。若事务T对数据对象A加上X锁，事务T可以读A也可以修改A，其他事务不能再对A加任何锁，直到T释放A上的锁。这保证了其他事务在T释放A上的锁之前不能再读取和修改A。

sql实现：在执行语句后加”for update”。

4.重建二叉树

题目描述：

输入某二叉树的前序遍历和中序遍历的结果，请重建出该二叉树。假设输入的前序遍历和中序遍历的结果中都不含重复的数字。例如输入前序遍历序列{1,2,4,7,3,5,6,8}和中序遍历序列{4,7,2,1,5,3,8,6}，则重建二叉树并返回。

8.跳台阶

题目描述：

一只青蛙一次可以跳上1级台阶，也可以跳上2级。求该青蛙跳上一个n级的台阶总共有多少种跳法（先后次序不同算不同的结果）。

思路：

假设跳n级有f(n)种跳法，考虑最后一步，最后一步要么跳一级，要么跳两级。最后一步跳一级有f(n-1)种跳法，最后一步跳两级有f(n-2)种跳法，所以f(n)=f(n-1)+f(n-2)。

9.变态跳台阶

题目描述：

一只青蛙一次可以跳上1级台阶，也可以跳上2级……它也可以跳上n级。求该青蛙跳上一个n级的台阶总共有多少种跳法。

思路：

思路同上，考虑最后一步。假设跳n级有f(n)种跳法，则f(n) = f(0)+f(1)+…+f(n-1)，那f(n-1)=f(0)+f(1)+…+f(n-2)，所以f(n) = 2*f(n-1)。

10.矩形覆盖

题目描述：

我们可以用2*1的小矩形横着或者竖着去覆盖更大的矩形。请问用n个2*1的小矩形无重叠地覆盖一个2*n的大矩形，总共有多少种方法？

思路：

思路同上，假设有f(n)种放法，考虑最后一块小矩形的放法，要么横着放，要么竖着放。最后一块横着放有f(n-2)放法，最后一块竖着放有f(n-1)种放法。

11.二进制中1的个数

思路：

15.反转链表

题目描述：输入一个链表，反转链表后，输出新链表的表头。

思路：从表头开始，首先用next保存当前节点head的下一节点head.next，不然就找不到链表其他节点了，然后让当前节点head作为新链表pre的表头，再让pre指向当前节点head，那pre就变成新链表的表头，再把之前保存的下一节点next赋值给当前节点head。不断循环，直到当前结点head为空。
