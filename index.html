1.类的生命周期

1.1加载

1）通过一个类的全限定名来获取定义此类的二进制字节流。

2）将这个字节流所代表的静态存储结构转化为方法区的运行时数据结构。

类文件结构

发表于 2018-09-21 | 分类于 JVM

1.魔数和Class文件版本

每个class文件前四个字节称为魔数，就是用来确定该文件是否能被虚拟机所接受的class文件。

第5和第6个字节表示次版本号，第7和第8字节表示主版本号。

2.常量池

紧接着主次版本号之后的就是常量池入口，可理解为class文件的资源库。主要用来存放两类常量：字面量和符号引用。字面量比较接近java语言层面的常量概念，如文本字符串、final修饰的常量等。而符号引用属于编译原理方面的概念，主要包括三类常量：类和接口的全限定名、字段的名称和描述符、方法的名称和描述符。

3.访问标志

常量池之后的两个字节代表访问标志，用于识别一些类或接口层次的访问信息，包括：这个Class是类还是接口；是否定义为public类型；是否定义为abstract类型；如果是类，有没有被final修饰等。

4.类索引、父类索引和接口索引集合

访问标志之后就是类索引、父类索引和接口索引集合。类索引用于确定这个类的全限定名，父类索引用于确定这个类的父类的全限定名，接口索引集合用于描述这个类实现哪些接口。

垃圾收集

发表于 2018-09-20 | 分类于 JVM

1.概述

垃圾收集针对的就是java堆和方法区。程序计数器、栈的内存分配和回收都具备确定性的，内存随着方法结束或线程结束而回收，就不用再考虑回收的问题了。而java堆和方法区不一样，例如一个接口的多个实现类需要的内存可能不一样，也就是说我们只有在程序运行时才知道创建哪些对象，所以这部分内存分配和回收都是动态不确定的，就需要我们去考虑回收的问题。

2.判断对象是否存活

主要有两种方式：引用计数法和可达性分析算法

2.1引用计算法

原理：给对象中添加一个引用计数器，每当一个地方引用它，计数器就加1；有一个引用失效，就减1；任何时刻只要计数器值为0就判定对象死亡。

弊端：解决不了对象间相互循环引用的问题。

2.1可达性分析算法

原理：通过一系列称为“GC Roots”的对象作为起始点，从这些节点开始向下搜索，搜索所走过的路径称为引用链，当一个对象到GC Roots没有任何引用链相连时，就说明该对象不可用。

JVM自动内存管理机制

发表于 2018-09-19 | 分类于 JVM

1.JVM运行时数据区域

1.1程序计数器

程序计数器可看作当前线程所执行的字节码的行号指示器，其实就是告诉JVM执行到字节码文件哪一行了。多线程环境中，每次线程切换后都能恢复到正确的执行位置，靠的就是这个计数器，这也解释了为什么程序计数器必须是线程私有的。另外，它也是唯一一个不会发生内存溢出异常的区域。

1.2虚拟机栈

就是为JVM执行java方法服务的。一个java方法执行的同时会创建一个栈帧用于保存局部变量表、操作数栈、动态链接、方法出口等信息。一个java方法从调用到执行结束，就对应着一个栈帧在虚拟机栈中入栈到出栈的过程。另外，如果线程请求的栈深度大于JVM所允许的深度，就会报StackOverFlowError；如果JVM可以动态扩展，但扩展时却申请不到足够的内存，就会报OutOfMemoryError。

乐观锁和悲观锁

发表于 2018-09-18 | 分类于 java面经

乐观锁：

概念：乐观锁是指操作数据库时(更新操作)，想法很乐观，认为每次的操作不会导致冲突，所以操作数据时不加锁，操作完成后，再判断是否有冲突。

实现：主要有版本号机制或CAS操作两种。

​ 1.版本号机制：前提是操作的数据库表有version字段，每次更新操作，都会将version字段值加1。一次完整的更新操作分为两步：（以更新状态为例）

s1.查出更新行信息，包含version字段。

select (status, version) from table where id = #{id};

s2.更新行信息，version字段加1。

update table set status = #{status}, version = version+1 where id=#{id} and version=#{version};

注：#{version}的值就是第一步查询出来的version值。

如果在更新期间有其他线程更新了该行，就会导致本次更新失败。

​ 2.CAS操作：即compare and swap 或者 compare and set，涉及到三个操作数，数据所在的内存值，预期值，新值。当需要更新时，判断当前内存值与之前取到的值是否相等，若相等，则用新值更新，若失败则重试，一般情况下是一个自旋操作，即不断的重试。
